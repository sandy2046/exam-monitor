# **考场大屏可视化系统 (ExamStage Monitor) 工业级全套开发包规格说明书 (V3)**

为了确保系统的绝对稳定性、高抗光干扰特性以及低CPU占用，本文档为 Coding Agent 规定了核心需求、技术栈、状态机及视觉规范。

## **1\. 需求规格说明书 (PRD) 深度扩展**

### **1.1 Master Clock 与硬件时钟微调机制**

* **绝对高频轮询**：放弃分散的 setInterval，采用由 requestAnimationFrame 驱动的每 16.67ms 统一主时钟引擎。杜绝多媒体电脑高负载运行或浏览器切换至后台标签页时的“丢帧与时间漂移”现象。  
* **时钟校准机制**：系统时钟公式设定为 DisplayTime \= HardwareTime \+ CalibrationOffset。允许在后端的配置控制台以秒为单位在 \[-300s, \+300s\] 范围内微调（前进或后退），大屏组件通过单向响应式数据流实时无缝重算渲染，绝对对齐考场物理铃声。

### **1.2 三段式高对比度大屏排版拓扑**

* **网格拓扑锁定**：严格限定大屏界面为顶部全局元数据(8%)、中部智能倒计时(52%)、中下双角色任务栏(32%)、底部线性进度轴(8%) 的固定网格，禁止任何形式的 z-index 悬浮、Toast 提示或弹出式 Modal，防止考场关键时间节点被意外遮挡。  
* **超远视距动态预警机制**：  
  * 正常答题期（剩余时间 \> 15分钟）：text-emerald-500 静态呈现。  
  * 尾声预警期（5分钟 \< 剩余时间 \<= 15分钟）：变更为 text-amber-500 并附带 Tailwind animate-pulse 呼吸动画（周期 2s）。  
  * 极端危急冲刺期（剩余时间 \<= 5分钟）：强制转换为 text-red-500 font-extrabold，且每逢绝对秒交替时通过 CSS transform: scale() 触发一次持续 150ms 的模拟心脏跳动微幅放大震慑效果，确保强光环境、幕布泛白或最后一排考生依然保持强烈的视觉警示。

## **2\. 技术栈定义与底层架构 (tech-stack.md)**

### **2.1 低开销运行约束 (Core Tech-Stack)**

* **技术核心选型**：锁死 Vue 3 (Composition API / \<script setup\>) 或 React 18。构建工具必须使用 Vite 5.x。  
* **零重型外部依赖**：严禁引入 Vue Router、React Router、Pinia、Redux 等外部中间件。整场考试配置与生命周期状态全量通过最轻量化的原生 reactive / useContext 维护，将老旧多媒体教学电脑（如常见 Core i3、4GB 内存）的 CPU 开销降到最低，杜绝因内存泄漏导致的黑屏或卡死。  
* **数字抖动消除**：样式纯 Tailwind CSS v3.x 编写。核心时钟与数字渲染必须强制开启 tabular-nums，使每个数字字符等宽，防止倒计时变动时因字符物理宽度不等造成整个组件的高频左右晃动。

### **2.2 绝对离线可用 (Offline-First) 与 PWA 规约**

* **缓存离线机制**：必须集成 vite-plugin-pwa 插件并配置 Service Worker 强缓存。在系统首次运行或加载后，对所有静态 \`.html\`、\`.js\`、\`.css\`、本地 SVG 图标及离线字体包进行持久化本地缓存。确保在开考后网络突发阻断甚至拔掉网线的情况下，大屏不仅文字不卡顿，即便考生刷新页面也能在完全离线的状态下秒级独立重新启动。

## **3\. 状态机流转与数据字典 (state-machine.md)**

### **3.1 考试确定性有限状态机 (Deterministic FSM)**

* **六大标准生命周期**：  
  * PREPARING：考前准备与组织入场（开考前 30 分钟 \- 开考前 15 分钟）  
  * DISTRIBUTING：材料清点与试卷分发（开考前 15 分钟 \- 开考前 0 分钟）  
  * EXAMING\_NORMAL：正常考试答题中（开考 0 分钟 \- 结束前 15 分钟）  
  * EXAMING\_WARNING：考试结束尾声预警（结束前 15 分钟 \- 结束前 5 分钟）  
  * EXAMING\_CRITICAL：极端临界冲刺期（结束前 5 分钟 \- 考试结束）  
  * EXAM\_FINISHED：考试结束及收卷整理（考试结束 \- 结束后 15 分钟）  
* **流转逻辑可靠性**：状态流转的推进仅依赖于“Master Clock 绝对时间”与“时间轴序列配置”的确定性交集映射。消除了因分布式网络延迟导致多台大屏阶段无法自动流转跳转的隐患。

### **3.2 灾难断电无缝恢复 (localStorage Resilience)**

* **故障恢复算法**：引擎在每秒时间心跳触发时，必须将当前执行的 examId、基准校准时间戳、当前激活的 Stage 快照写入浏览器的 localStorage。当考场发生突然停电、主机死机、或操作员误关闭浏览器等灾难性事件时，重新打开网页会自动计算时间差值，实现 **0秒无人干预自动恢复**，并保持状态机步调一致。

## **4\. UI/UX 视觉大屏样式字典 (ui-style-guide.md)**

### **4.1 投影仪抗光干扰色彩令牌 (Design Tokens)**

* **深色高反差模式 (Dark Mode \- 默认推荐)**：底色锁定 bg-\[\#090D16\]（高饱和暗夜蓝），主要文字 text-\[\#F8FAFC\]（视网膜级白），卡片底色 bg-\[\#131C2E\]。  
* **浅色高反差模式 (Light Mode)**：底色 bg-\[\#FFFFFF\]，主要文字 text-\[\#0F172A\]（深板岩黑），卡片 bg-\[\#F1F5F9\]。允许一键切置，防止白天考场强光导致大屏泛白无法识别。

### **4.2 线性时间轴 SVG 渲染规范**

* **平滑渲染规范**：进度条轨道（Track）使用 SVG \<rect\> 元素。已流逝进度（Fill）绑定通过 Master Clock 重算后的宽度百分比，填充高性能渐变色。当前时间游标（Indicator）集成外发光滤镜（filter="url(\#glow)"），通过 transform: translateX() 矩阵变换由引擎直接驱动，保证像素级别的平滑无缝位移。
